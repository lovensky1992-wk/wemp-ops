# 正文配图 Prompt 模板

用于公众号文章正文中的信息图、概念图配图。封面图见 cover-image-guide.md。

## 风格一：Notion 手绘信息图（推荐，适合概念解释、流程说明）

```
# 角色
你是一位擅长 Notion 官方插画风格的信息可视化设计师，能将任何内容转化为松弛感线稿信息图。

# 视觉风格规范
## 线条特征
- 以单一主色为主（根据文章主题选择：科技蓝 #4A90D9、活力橙 #FF6B35、极客绿 #2ECC71）
- 线条粗细不均匀，像马克笔随手画的质感
- 笔触松弛、略带抖动，不追求工整
- 所有图标、人物、图表都保持手绘涂鸦感
- 少部分地方填充深色让整体压实

## 人物画法（如需要）
- 简笔画风格：圆润的头、点或线表示五官
- 肢体夸张、动作松弛自然
- 可以是完整人物或只露出局部（手、上半身）

## 通用禁止事项
- 不要彩色渐变或复杂配色
- 不要粗黑边框或生硬分隔线
- 不要 3D 效果、阴影、立体感
- 不要密集文字堆砌，文字内容简化为核心关键词（3-5字）
- 保持大量留白和呼吸感
- 装饰元素要克制

# 输出
- 直接生成符合上述风格的信息图，中文，不需要解释。
- 图片比例 16:9

# 内容
[在此描述要可视化的内容]
```

### 适用场景
- 文章中间的概念解释图
- 对比分析图（A vs B）
- 功能要点汇总
- 工作流程示意

## 风格二：扁平科技信息图（适合数据对比、技术架构）

```
极简扁平化科技信息图。
配色限定为深蓝(#1E3A5F)、电光蓝(#3B82F6)、纯白、浅灰(#F1F5F9)。
线条简洁有力(2-3px均匀粗细)，造型几何化干净，纯色填充，零纹理。
白色背景，无阴影，无渐变，高对比度。
中文文字采用无衬线字体，仅提取核心关键词（3-5字），绝不大段文字。
图标使用线性简笔风格，保持统一视觉语言。
整体风格：专业、克制、信息密度适中。
图片比例 16:9。

内容：[在此描述要可视化的内容]
```

### 适用场景
- 技术架构图
- 性能/价格对比
- 产品功能矩阵
- 时间线/路线图

## 风格三：终端/编辑器截图风（适合教程、实战类）

不使用 AI 生图，而是**真实截图 + 美化**。

### 获取方式
1. 实际操作并截图（终端命令、编辑器界面、产品界面）
2. 用截图美化脚本处理：`~/.openclaw/workspace/scripts/beautify-screenshot.sh`
3. 输出带圆角、阴影、纯色背景的美化截图

### 适用场景
- 代码演示
- 产品界面展示
- 操作步骤教程
- 效果对比（before/after）

---

## 使用决策树

```
文章内容类型判断：
├─ 概念解释 / 流程说明 / 要点汇总 → 风格一（Notion 手绘）
├─ 数据对比 / 技术架构 / 产品矩阵 → 风格二（扁平科技）
├─ 实际操作 / 教程步骤 / 产品展示 → 风格三（真实截图）
└─ 都不是 / 内容简单 → 不配图
```

## 配图数量控制

- 公众号文章：正文 2-4 张，宁缺毋滥
- 信息图不要每个小节都配，选最有价值的 2-3 个点
- 真实截图不限数量，但避免连续放 3 张以上无解释的截图

## 生图注意事项

- 使用 nano-banana-pro skill 生成
- 同一篇文章的多张配图在**同一次对话**中生成，保持风格一致
- 生成后检查中文文字是否清晰可读，不清晰则重新生成
- 去水印：生成后用 `~/.openclaw/workspace/scripts/remove-watermark.sh` 处理
